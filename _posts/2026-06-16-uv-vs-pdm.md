# uv vs PDM: Modern Python Project Management

While setting up a new portfolio project, I decided to move away from a traditional `pip + requirements.txt` workflow and look into modern Python project managers. Two tools that came up repeatedly were **uv** and **PDM**.

This post summarizes what I learned while comparing them.

## Why use a project manager at all?

For a long time, my Python workflow looked roughly like this:

```powershell
python -m venv .venv
pip install -r requirements.txt
```

This works, but as projects grow it becomes harder to manage:

* dependencies and dependency versions
* virtual environments
* development dependencies
* reproducible project setup
* packaging and publishing

Modern tools such as uv and PDM try to solve these problems in a more integrated way.

## What is uv?

- uv is a Python package and project manager for fast package installation and dependency resolution.
- It has built-in virtual environment management.
- Lock files ensure reproducability (see uv.lock)
- CLI tools can be run using uvx without the need for permanent installation


Installation: 

```powershell
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Useful Commands:

Initialize projects:
```powershell
uv init 
```

Add dependencies

```powershell
uv add pandas
```

Run scripts without virtual environment activation

```powershell
uv run python script.py
```

Sync the environment (explicitly)

```powershell
uv sync
```



## What is PDM?

PDM is another modern Python package and project manager for dependency and virtual environment management.

Installation: 
	
```powershell
powershell -ExecutionPolicy ByPass -c "irm https://pdm-project.org/install.ps1 | iex"
```

Useful Commands:

Create a new project or create a pyproject.toml for an existing project
```powershell
pdm new my-project
pdm init
````
Adding dependencies
```powershell
pdm add pandas
```
Install dependencies from lock file
```powershell
pdm install
```

Run scripts without virtual environment activation

```powershell
pdm run python script.py
```

## Similarities

Both tools:

* use `pyproject.toml`
* manage dependencies
* support reproducible environments
* generate lockfiles
* provide command runners
* aim to replace parts of older workflows involving pip, virtualenv, and requirements.txt

## Differences

### uv

Pros:

* very fast
* simple workflow
* growing adoption
* good developer experience

Potential drawbacks:

* relatively young project
* ecosystem still evolving

### PDM

Pros:

* mature standards-based approach
* strong packaging support
* flexible project configuration

Potential drawbacks:

* slower than uv
* slightly more complexity for simple projects
