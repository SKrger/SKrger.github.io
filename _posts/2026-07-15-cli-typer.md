Command-line tools are a great way to automate workflows, but as they grow, a single script can quickly become difficult to maintain. **Typer** makes it simple to build modern Python CLIs using type hints, automatic help generation, and clean command organization.

In this post, we'll build a small modular CLI and turn it into a real command-line application that can be launched with a custom command such as `greeting` instead of `python main.py`.

<!-- more -->

## Why Typer?

Traditional CLI libraries often require manually parsing arguments and writing help text. Typer uses Python type hints to generate much of this automatically.

A simple Typer command looks like this:

```python
import typer

app = typer.Typer()

@app.command()
def hello(name: str):
    """Say hello to someone."""
    print(f"Hello, {name}!")

if __name__ == "__main__":
    app()
```

Running:

```bash
python main.py Alice
```

produces:

```
Hello, Alice!
```

Typer also automatically creates documentation:

```bash
python main.py --help
```

## Organizing Commands into Modules

As a CLI grows, it is better to split commands into separate modules. For example:

```text
greeting_cli/
├── __init__.py
├── main.py
├── users.py
└── projects.py
```

Each module owns a group of related commands.

### users.py

```python
import typer

app = typer.Typer()

@app.command()
def list():
    """List all users."""
    print("Alice")
    print("Bob")
```

### projects.py

```python
import typer

app = typer.Typer()

@app.command()
def create(name: str):
    """Create a project."""
    print(f"Created project: {name}")
```

The main application combines these modules:

### main.py

```python
import typer

from greeting_cli.users import app as users_app
from greeting_cli.projects import app as projects_app

app = typer.Typer()

app.add_typer(users_app, name="users")
app.add_typer(projects_app, name="projects")

if __name__ == "__main__":
    app()
```

Now the CLI has clean subcommands:

```bash
python main.py users list
```

Output:

```
Alice
Bob
```

and:

```bash
python main.py projects create website
```

Output:

```
Created project: website
```

## Creating Your Own CLI Command

Typing `python main.py` every time is fine during development, but a polished CLI should have its own command.

Python packaging makes this easy with a console script entry point.

Add the following sections to `pyproject.toml`. This creates a command called `greeting` that runs your Typer application.

```toml

[project.scripts]
greeting = "greeting_cli.main:app"

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["greeting_cli"]

[tool.uv]
package = true
```


Install it locally:

```bash
pip install -e .
```

or if you are using uv 

```bash
uv sync
```

Now you can use:

```bash
greeting users list
```

instead of:

```bash
python main.py users list
```

Typer's help system works exactly as before:

```bash
greeting --help
greeting users --help
greeting projects --help
```

## Conclusion

Typer provides a simple path from a small Python script to a fully structured command-line application. By splitting commands into modules and exposing the application through a console script, you get a CLI that is easy to maintain and feels like a native tool.

A well-designed Typer application can start as a few lines of code and grow into a powerful developer tool without becoming difficult to manage.
